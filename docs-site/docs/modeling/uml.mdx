## Диаграмма последовательностей

Диаграмма последовательностей описывает процесс загрузки публикации, её асинхронной обработки и получения результата пользователем

```plantuml
@startuml
title Обработка публикации и просмотр результата

actor "Химик-исследователь" as User
participant "Web UI" as UI
participant "Backend/API" as API
queue "Очередь задач" as Queue
participant "Сервис обработки" as Processing
participant "ML/NLP Extractor" as Extractor
database "База данных" as DB

== Загрузка публикации ==

User -> UI: Загрузить публикацию
UI -> API: Передать файл и метаданные

API -> DB: Сохранить публикацию\nсо статусом "загружена"
DB --> API: publication_id

API -> Queue: Поставить задачу на обработку\npublication_id
API --> UI: Подтвердить загрузку\nи показать статус "загружена"
UI --> User: Показать сообщение:\n"Публикация загружена и поставлена в обработку"

== Асинхронная обработка ==

Queue -> Processing: Передать задачу на обработку\npublication_id

Processing -> API: Запросить публикацию для обработки\npublication_id
API -> DB: Получить файл, метаданные\nи текущий статус публикации
DB --> API: Данные публикации
API --> Processing: Передать данные публикации

alt Файл валиден
    Processing -> Extractor: Извлечь данные из публикации\nфайл + метаданные

    alt Данные извлечены успешно
        Extractor --> Processing: Результат извлечения\nstatus = success

        Processing -> API: Передать результат обработки\nкарточка катализатора + status = success
        API -> DB: Сохранить карточку катализатора
        API -> DB: Обновить статус публикации\n"обработано"

    else Данные извлечены частично
        Extractor --> Processing: Частичный результат\nwarnings

        Processing -> API: Передать частичный результат\nнеполная карточка + warnings
        API -> DB: Сохранить неполную карточку
        API -> DB: Обновить статус публикации\n"обработано с предупреждением"

    else Ошибка извлечения
        Extractor --> Processing: Ошибка извлечения

        Processing -> API: Передать ошибку обработки
        API -> DB: Обновить статус публикации\n"ошибка обработки"
    end

else Файл невалиден
    Processing -> API: Передать ошибку валидации
    API -> DB: Обновить статус публикации\n"ошибка валидации"
end

== Просмотр результата ==

User -> UI: Посмотреть результат обработки
UI -> API: Запросить статус и результат\npublication_id

API -> DB: Получить статус публикации\nи результат обработки
DB --> API: Статус + результат / ошибка / предупреждения

API --> UI: Вернуть статус и результат обработки
UI --> User: Показать результат обработки

@enduml
```

### Описание

Основные этапы:

- Пользователь загружает публикацию через Web UI
- Backend сохраняет публикацию и ставит задачу в очередь
- Сервис обработки извлекает данные (ML/NLP extractor)
- Формируется карточка катализатора
- Результат сохраняется в базе данных
- Пользователь может запросить статус и результат обработки

### Особенности

- Асинхронная обработка через очередь задач
- Возможность частичного результата
- Обработка ошибок

## Диаграмма вариантов использования

Диаграмма вариантов использования описывает основные сценарии взаимодействия пользователей с системой

```plantuml
@startuml
left to right direction

actor "Химик-исследователь" as Researcher
actor "Администратор" as Admin

rectangle "Система извлечения данных из научных публикаций" {
    usecase "Загрузить публикацию" as UC_Upload
    usecase "Извлечь данные из публикации" as UC_Extract
    usecase "Сформировать карточку катализатора" as UC_CreateCard

    usecase "Просмотреть базу данных карточек" as UC_ViewDB
    usecase "Просмотреть карточку" as UC_ViewCard
    usecase "Отредактировать карточку" as UC_EditCard

    usecase "Управлять пользователями" as UC_ManageUsers
}

Researcher --> UC_Upload
Researcher --> UC_ViewDB

Admin --> UC_ManageUsers

UC_Upload ..> UC_Extract : include
UC_Extract ..> UC_CreateCard : include

UC_ViewCard ..> UC_ViewDB : extend
UC_EditCard ..> UC_ViewCard : extend

@enduml
```

### Акторы

- **Химик-исследователь**
- **Администратор**

### Основные сценарии

Для химика:

- Загрузка публикации
- Просмотр карточки катализатора
- Редактирование карточки
- Просмотр базы данных

Для администратора:

- Управление пользователями

### Внутренние процессы системы

- Извлечение данных из публикации
- Формирование карточки катализатора
