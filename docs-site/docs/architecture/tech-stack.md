# Технологический стек

| Компонент | Технология | Назначение |
|---|---|---|
| Frontend | Docusaurus / Markdown | Документация и прототип интерфейса |
| API | REST API | Синхронное взаимодействие между frontend и backend |
| Асинхронная обработка | RabbitMQ | Очередь задач и результаты обработки публикаций |
| Обработка данных | Processing Service | Извлечение метаданных из публикаций |
| База данных | PostgreSQL | Хранение структурированных данных |
| Хранилище файлов | Object Storage | Хранение PDF и изображений |
| Документация API | OpenAPI / AsyncAPI | Описание REST и асинхронных взаимодействий |
| Диаграммы | PlantUML | Визуализация архитектуры |

---

:::note
В системе используется разделение на синхронную (REST API) и асинхронную (RabbitMQ) части для обработки длительных операций
:::
