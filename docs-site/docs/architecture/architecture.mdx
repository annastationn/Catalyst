# Архитектура

## C1 — Контекстная диаграмма

Диаграмма показывает систему обработки публикаций с участием внешних участников и сервисов 

```plantuml
@startuml
!includeurl https://raw.githubusercontent.com/plantuml-stdlib/C4-PlantUML/master/C4_Context.puml

LAYOUT_TOP_DOWN()

title System Context diagram — Catalyst Processing System

Person(researcher, "Химик-исследователь", "Исследователь, загружающий публикации и анализирующий карточки катализаторов")
Person(admin, "Администратор", "Управляет данными и контролирует процесс обработки")

System(catalystSystem, "Catalyst Processing System", "Система обработки научных публикаций и формирования карточек катализаторов")

System_Ext(storage, "Object Storage", "Хранение PDF-файлов публикаций и изображений химических структур")
SystemDb_Ext(database, "PostgreSQL", "Хранение публикаций, карточек катализаторов, данных о синтезах и статусах обработки")

Rel(researcher, catalystSystem, "Загружает публикации и просматривает результаты")
Rel(admin, catalystSystem, "Редактирует данные и контролирует обработку")

Rel(catalystSystem, storage, "Сохраняет и извлекает PDF-файлы и изображения")
Rel(catalystSystem, database, "Читает и записывает структурированные данные")

@enduml
```


## C2 — Контейнерная диаграмма
Диаграмма показывает внутреннее взаимодействий компонентов системы, демонстрирует связи между ними

```plantuml
@startuml
!include https://raw.githubusercontent.com/plantuml-stdlib/C4-PlantUML/master/C4_Container.puml

LAYOUT_TOP_DOWN()

title Container diagram — Catalyst Processing System

Person(researcher, "Химик-исследователь", "Загружает научные публикации, просматривает архив и карточки катализаторов")
Person(admin, "Администратор", "Управляет данными и контролирует процесс обработки")

System_Boundary(catalystSystem, "Catalyst Processing System") {

    Container(webUi, "Web UI", "Frontend", "Интерфейс загрузки публикаций, просмотра архива и карточек катализаторов")

    Container(backendApi, "Backend/API", "REST API", "Управление публикациями, карточками катализаторов, пользователями и статусами обработки")

    ContainerQueue(taskQueue, "Очередь задач", "Message Queue", "Очередь задач асинхронной обработки публикаций")

    Container(processingService, "Processing Service", "Backend Worker", "Обработка публикаций, управление пайплайном извлечения данных")

    Container(mlExtractor, "ML/NLP Extractor", "ML/NLP Service", "Извлечение параметров синтеза и данных о катализаторах из текста публикаций")

    ContainerDb(database, "PostgreSQL", "Relational Database", "Структурированные данные: публикации, карточки катализаторов, синтезы, статусы обработки")

    Container(storage, "Object Storage", "File Storage", "PDF-файлы публикаций и изображения химических структур")
}

Rel(researcher, webUi, "Работает с интерфейсом")
Rel(admin, webUi, "Управляет данными через интерфейс")

Rel(webUi, backendApi, "Отправляет запросы", "REST API")

Rel(backendApi, database, "Читает и записывает данные")
Rel(backendApi, storage, "Сохраняет и получает PDF-файлы и изображения")
Rel(backendApi, taskQueue, "Создаёт задачу обработки", "Message")

Rel(taskQueue, processingService, "Передаёт задачу обработки", "Message")

Rel(processingService, backendApi, "Запрашивает данные публикации, передаёт результат обработки и обновляет статус", "REST API")
Rel(processingService, storage, "Получает файл публикации")
Rel(processingService, mlExtractor, "Передаёт текст и метаданные для извлечения")
Rel(mlExtractor, processingService, "Возвращает извлечённые данные")

@enduml
```

## Основные компоненты

| Компонент | Назначение |
|---|---|
| Frontend | Интерфейс для загрузки публикаций, просмотра архива, поиска и взаимодействия с карточками катализаторов |
| API Service | Основной backend, который принимает REST-запросы, работает с БД и отправляет задачи на обработку |
| Processing Service | Сервис обработки публикаций и извлечения данных о катализаторах|
| RabbitMQ | Брокер сообщений для асинхронной передачи задач и результатов обработки |
| PostgreSQL | Реляционная БД для хранения метаданных|
| Object Storage | Хранилище PDF публикаций и изображений структур катализаторов |

## Внешние зависимости

| Сервис | Тип интеграции | Описание |
|---|---|---|
| Object Storage | HTTP / SDK | Хранение PDF публикаций и изображений структур|
| RabbitMQ | AMQP | Асинхронная передача задач обработки и результатов |
| PostgreSQL | SQL | Хранение публикаций, катализаторов, синтезов, пользователей |
