# Архитектура

## C1 — Контекстная диаграмма

Диаграмма показывает систему обработки публикаций с участием внешних участников и сервисов 

```plantuml
@startuml

title C1 — Системная контекстная диаграмма
actor "Химик-исследователь" as researcher
actor "Администратор" as admin

rectangle "Catalyst Processing System\n\nСистема обработки научных публикаций\nи формирования карточек катализаторов" as system
rectangle "Object Storage\n\nPDF-файлы и изображения структур" as storage
database "PostgreSQL\n\nПубликации, карточки катализаторов,\nсинтезы и статусы обработки" as db

researcher --> system : загружает публикации\nпросматривает результаты
admin --> system : редактирует данные\nконтролирует обработку
system --> storage : сохраняет и получает файл
system --> db : читает и записывает данные

@enduml
```


## C2 — Контейнерная диаграмма
Диаграмма показывает внутреннее взаимодействий компонентов системы, демонстрирует связи между ними

```plantuml
@startuml

title C2 — Контейнерная диаграмма
actor "Химик-исследователь" as researcher
actor "Администратор" as admin
rectangle "Catalyst Processing System" {
  rectangle "Web UI\n\nИнтерфейс загрузки публикаций,\nпросмотра архива и карточек" as frontend
  rectangle "Backend/API\n\nREST API, управление публикациями,\nкарточками и статусами" as api
  queue "Очередь задач\n\nЗадачи асинхронной обработки" as queue
  rectangle "Processing Service\n\nОбработка публикаций,\nзапуск извлечения данных" as processing
  rectangle "ML/NLP Extractor\n\nИзвлечение параметров синтеза\nи данных о катализаторах" as extractor
  database "PostgreSQL\n\nСтруктурированные данные" as db
  rectangle "Object Storage\n\nPDF-файлы и изображения" as storage
}

researcher --> frontend : работает с интерфейсом
admin --> frontend : управляет данными
frontend --> api : REST API
api --> db : SQL-запросы
api --> storage : сохраняет PDF\nи изображения
api --> queue : создает задачу обработки
queue --> processing : передает задачу
processing --> storage : получает файл публикации
processing --> extractor : передает текст\nи метаданные
extractor --> processing : возвращает извлеченные данные
processing --> db : сохраняет результат
processing --> queue : возвращает статус обработки

@enduml
```

## Основные компоненты

| Компонент | Назначение |
|---|---|
| Frontend | Интерфейс для загрузки публикаций, просмотра архива, поиска и взаимодействия с карточками катализаторов |
| API Service | Основной backend, который принимает REST-запросы, работает с БД и отправляет задачи на обработку |
| Processing Service | Сервис обработки публикаций и извлечения данных о катализаторах|
| RabbitMQ | Брокер сообщений для асинхронной передачи задач и результатов обработки |
| PostgreSQL | Реляционная БД для хранения метаданных|
| Object Storage | Хранилище PDF публикаций и изображений структур катализаторов |

## Внешние зависимости

| Сервис | Тип интеграции | Описание |
|---|---|---|
| Object Storage | HTTP / SDK | Хранение PDF публикаций и изображений структур|
| RabbitMQ | AMQP | Асинхронная передача задач обработки и результатов |
| PostgreSQL | SQL | Хранение публикаций, катализаторов, синтезов, пользователей |
