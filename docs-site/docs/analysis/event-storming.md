# Event Storming

В данном разделе представлена модель построенная с использованием метода Event Storming

![Event Storming](/img/event_storming.png)

---

## Описание процесса

Диаграмма отражает полный жизненный цикл обработки научной публикации:

1. Загрузка публикации пользователем
2. Создание задачи обработки
3. Постановка задачи в очередь
4. Асинхронная обработка
5. Извлечение текста (включая OCR)
6. Извлечение параметров синтеза (ML/NLP)
7. Формирование карточки катализатора
8. Сохранение результата
9. Завершение обработки

---

## Основные события 

- Публикация загружена
- Задача обработки создана
- Задача поставлена в очередь
- Обработка начата
- Текст извлечен
- Параметры синтеза извлечены
- Карточка сформирована
- Карточка сохранена
- Обработка завершена

---

## Команды 

- Загрузить публикацию
- Создать задачу обработки
- Поставить задачу в очередь
- Запустить обработку
- Извлечь текст
- Извлечь параметры синтеза
- Сформировать карточку
- Сохранить карточку

---

## Агрегаты и модели

- **Publication** — хранение загруженной публикации
- **ProcessingJob** — задача обработки
- **CatalystCard** — карточка катализатора

---

## Внешние системы

- OCR-сервис — используется при невозможности извлечения текста напрямую
- ML/NLP extractor — извлечение параметров синтеза

---

## Проблемные зоны 

- Потеря информации при OCR
- Некорректное извлечение параметров
- Нормализация единиц измерения
- Ошибки сохранения в БД
