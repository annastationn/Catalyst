Требования к системе

Функциональные требования​

Управление публикациями​

  • Система обеспечивает загрузку научных публикаций в формате PDF
  • Система сохраняет метаданные статьи
  • Система предоставляет список загруженных публикаций

Обработка публикаций​

  • Система запускает процесс обработки публикации
  • Система извлекает структурированные данные из текста
  • Система определяет информацию о катализаторах и условиях синтеза/остальных параметров

Обработка результатов​

  • Система сохраняет результаты обработки
  • Система предоставляет доступ к результатам через API
  • Система отображает результаты пользователю

Работа с API​

  • Система предоставляет REST API для доступа к данным
  • Система поддерживает операции создания, чтения и обновления данных
  • Система возвращает ответ

Нефункциональные требования​

Производительность​

  • Система должна обрабатывать публикации в разумное время
  • Система должна поддерживать параллельную обработку запросов

Надежность​

  • Система должна корректно обрабатывать ошибки
  • Система должна обеспечивать сохранность данных
  • Система должна быть устойчива к сбоям отдельных компонентов

Масштабируемость​

  • Система должна поддерживать горизонтальное масштабирование
  • Система должна позволять увеличивать нагрузку без изменения архитектуры

Безопасность​

  • Система должна обеспечивать аутентификацию пользователей
  • Система должна ограничивать доступ к данным

Поддерживаемость​

  • Система должна иметь модульную архитектуру
  • Система должна быть удобна для доработки и расширения

Ограничения​

  • Система работает с публикациями в формате PDF
  • Качество извлечения данных зависит от структуры документа
  • Система не гарантирует 100% точность извлечения
