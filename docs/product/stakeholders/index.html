Сбор требований

Цель этапа​

На этапе сбора требований были определены основные пользователи системы, их потребности, ожидания от продукта и сценарии

Основные стейкхолдеры​

УчастникИнтерес
Химик-сследовательБыстро получать структурированные данные из публикаций
Руководитель направленияОбработка данных и налаживание производства в промышленности
АдминистраторКонтролировать данные и корректировать карточки
Бизнес-заказчикСократить ручную работу с научными источниками
Команда разработчиковРеализация и настройка научного функционала

Основные потребности пользователей​

  • загрузка научных публикаций
  • автоматическое извлечение данных
  • просмотр найденных катализаторов и синтезов
  • поиск по базе карточек
  • редактирование найденных данных
  • хранение истории обработанных публикаций

Основные сценарии​

  1. Пользователь загружает публикацию
  2. Система запускает обработку
  3. Пользователь получает результат анализа
  4. Пользователь открывает карточки катализаторов и синтезов
  5. Пользователь при необходимости редактирует данные
